这是一个离线渲染器,实现了大量的光线追踪主流技术比如多重重要性采样, 启发式多重采样等.

Progressive Rendering和其他调试编辑器

离线渲染通常需要更长的时间,所以为了方便调试和预览,在多线程渲染的同时,我还支持了实时的更新当前已经渲染的结果到编辑器中.这需要在设计上保证渲染结果不停的基于之前的结果进行叠加.

BVH加速数据结构

使用BVH进行场景分割,以加速Ray Intersection的效率.同时BVH使用了启发式构建来增加构建速度.

Differential Ray贴图采样

通过发射differential ray来计算屏幕空间的UV变化梯度,来提高贴图采样的精细度.如下第一张图是简单的bilinear sampling,第二个是使用了differential ray来贴图采样.

启发式Ray Sampling

根据屏幕空间颜色变化的梯度,来决定某个像素的采样数量,来提高渲染效率,第二张图的灰度代表采样数量,越接近白色代表采样数越高,黑色区域都是颜色变化较少的区域,减少了计算的浪费.

mo-gfx, api-agnostic graphics api

这是一个中间件项目,主要的设计目标就是提供一个易用的更简洁的高层图形API抽象.我们现在面临的问题是,目前下一代的Api比如Dx12和Vulkan都过于的显式,开发效率极低,需要处理的问题很多,比如内存管理和显卡CPU之间的同步处理.如Vulkan,画出第一个三角形就需要我们一千五百行左右的实现.但是我们想做RTX光线追踪项目,这是我们绕不开的图形API.结合自己的需求进行抽象和封装就变成了一种必然.

目前项目进度大概是,我已经完成了Uniform, Pipeline, Buffer, Shader这几个核心对象的抽象.利用Handle Pool来方便开发者完全使用值使用各种渲染资源.同时屏蔽了Descriptor Pool, Command Buffer等细节.因为我的目标是易用性开发效率最优先,而不是抠极限的性能,所以才可以做到这种级别的简化.目前一个三角形的绘制大概只有如下的代码:
